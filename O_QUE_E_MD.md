# O Que É .md? Quais Fazer Upload?

Você viu menção a "todos os `.md`" e ficou confuso. Este guia explica o que são e quais você precisa fazer upload.

---

## 📄 O Que É `.md`?

`.md` é a extensão de arquivo **Markdown**. É um formato de texto simples usado para documentação.

**Exemplos de arquivos `.md`:**
- `README.md` - Documentação principal do projeto
- `DEPLOYMENT.md` - Guia de deployment
- `SETUP_IPHONE_ONLY.md` - Guia para iPhone
- `design.md` - Design do app
- `todo.md` - Lista de tarefas

## 📱 Como Fazer Upload dos `.md`

### Método 1: Safari

1. Abra Safari
2. Vá para seu repositório no GitHub
3. Clique em **Add file** → **Upload files**
4. Selecione TODOS os arquivos `.md` de uma vez
5. Clique em **Commit changes**

### Método 2: GitHub Mobile

1. Abra GitHub Mobile
2. Vá para seu repositório
3. Clique em **+** → **Upload files**
4. Selecione TODOS os `.md`
5. Clique em **Commit**

## 💡 Por Que Fazer Upload dos `.md`?

Estes arquivos são **documentação importante** que ajuda:

1. **Você** - A lembrar como fazer as coisas
2. **Outras pessoas** - A entender o projeto
3. **Render** - A entender como fazer deploy
4. **GitHub** - A mostrar informações do projeto

Quando alguém acessa seu repositório no GitHub, vê estes arquivos e entende:
- O que é o projeto
- Como fazer deploy
- Como usar o app
- Qual é o design
- Quais tarefas faltam fazer

## 🚀 Próximo Passo

Depois de fazer upload de TUDO (pastas + arquivos `.md`):

1. Abra [render.com](https://render.com)
2. Clique em **New +** → **Web Service**
3. Conecte seu repositório GitHub
4. Configure as variáveis de ambiente
5. Clique em **Create Web Service**

Render vai fazer deploy automaticamente! 🎉

---

## 💡 Dica Final

**Regra simples:**
- Se o arquivo termina em `.md` → ✅ Fazer upload
- Se é uma pasta (`app/`, `server/`, etc.) → ✅ Fazer upload
- Se é `node_modules/`, `.expo/`, `dist/` → ❌ NÃO fazer upload
